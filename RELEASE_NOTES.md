# CPA Auth Pool 0.1.21

This release fixes dynamic auth pools that displayed healthy accounts in CPA-Helper but returned `503 bound auth pool has no eligible auth candidates` at request time.

## Fixes

- Accept and persist `resolved_auth_ids` generated by CPA-Helper for dynamic account types such as `k12`, `team`, and `plus`.
- Match resolved account IDs as explicit pool members even when CLIProxyAPI candidates do not contain plan or tier metadata.
- Synchronize resolved membership independently from model catalogs so one model lookup failure cannot block pool membership recovery.
- Preserve last-good model catalogs during membership-only updates.
- Round-robin accounts with the same priority instead of repeatedly selecting the lexicographically first account.
- Save plugin state through a detached deep copy and atomic file replacement.
